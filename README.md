# 메디힘 전략 라이브러리 · 내부 공유 패키지

## 1. 파일 구성

- `index.html` : 접속번호 입력 후 문서 카드 허브가 열리는 메인 게이트 페이지
- `가설_종류_및_현업_적용.html` : 가설 프레임워크 본문 웹 문서
- `가설_종류_및_현업_적용.md` : Markdown 원문
- `가설_종류_및_현업_적용.docx` : Word 문서

## 2. 게이트 페이지 구조

1. 접속번호 입력
2. 메디힘 전략 라이브러리 진입
3. 카테고리 또는 검색어로 문서 탐색
4. 개별 프레임워크 카드에서 본문 문서 진입

현재 접속번호는 `3370`입니다. 기존 `0220` 인증값과 이전 세션 키는 제거되었습니다.

## 3. 새 문서 카드 추가 방법

`index.html` 하단 스크립트의 `documents` 배열에 아래 형태의 객체를 추가합니다.

```javascript
{
  title: '새 문서 제목',
  categories: ['사업전략', '운영'],
  description: '문서가 다루는 문제와 활용 범위를 설명합니다.',
  note: '카드 하단 행동 문구',
  meta: ['8개 섹션', '2026.07 업데이트'],
  href: '새_문서.html',
  status: '운영 중'
}
```

- 카드 번호는 배열 순서대로 자동 생성됩니다.
- `categories`에 입력한 값은 상단 카테고리 필터에 자동 추가됩니다.
- 검색은 제목, 설명, 보조 문구, 카테고리, 메타 정보를 대상으로 작동합니다.

## 4. HTML 배포 방법

1. 모든 파일을 동일한 폴더에 유지합니다.
2. GitHub 저장소에 폴더 전체를 업로드합니다.
3. GitHub Pages의 진입 파일은 `index.html`입니다.
4. 새 문서가 추가되면 HTML 파일을 같은 폴더에 넣고 `documents` 배열에 카드 정보를 추가합니다.

## 5. 유의사항

정적 HTML의 접속번호 게이트는 접근 편의를 위한 장치이며 강한 보안 기능이 아닙니다. 민감한 사내 정보는 서버 인증, 사내 드라이브 권한 또는 접근제어가 가능한 문서 시스템을 사용하십시오.


## Pretendard 폰트 적용

- HTML: Pretendard Variable 공식 CDN을 연결하고 전체 UI의 기본 서체로 적용
- Word: 문서 스타일과 본문·표·머리글·바닥글을 Pretendard로 통일
- 웹폰트 연결 실패 시 시스템 한글 서체로 자동 대체
