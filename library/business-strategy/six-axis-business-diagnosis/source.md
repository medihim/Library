# 1. 핵심 결론

메디힘의 현재 사업은 **시장 선택과 고객 문제 이해는 비교적 명확하지만, 성공 사례를 여러 병원에서 반복할 수 있는 ‘표준화된 성장 시스템’으로 전환하는 단계**에 있습니다.

현재 가장 큰 병목은 시장 수요가 아니라 다음 세 가지입니다.

1. **신규 환자 획득의 재현성**
2. **상담에서 예약·매출까지의 전환 표준화**
3. **인력 투입량과 무관하게 이익이 증가하는 경제성 구조**

종합점수는 **20점/30점**으로 평가할 수 있습니다.

> **진단:** 시장·고객 적합성은 양호하지만, 아직 확장 가능한 사업모델이 완성된 상태는 아니다.  
> **현재 단계:** 초기 사업 검증 이후, 재현성·제품화·단위경제성을 검증하는 단계

# 2. 메디힘 사업에 대입한 최종 6문장

## 2.1 시장

메디힘은 국내 환자만으로는 지속적인 성장이 어려운 병원과, 한국 의료서비스를 원하지만 정보 부족과 불확실성으로 결정을 미루는 해외환자의 문제가 향후에도 커질 것이라는 시장가설을 선택했다.

## 2.2 고객

병원은 단순한 광고 집행이 아니라 해외환자의 문의·상담·예약·내원·매출이 반복되는 구조를 필요로 하고, 해외환자는 병원 선택, 의료진 신뢰, 가격, 통역, 사후관리의 불확실성을 해결할 수 있는 일관된 의사결정 경험을 필요로 한다.

## 2.3 표본

메디힘은 일본 직접 마케팅, 중국 에이전시, 영어권 채널, 비대면 상담회 등의 시장 진입 경로를 확보하고 있으나, 특정 병원이나 특정 시기에 의존하지 않고 신규 환자가 지속적으로 유입되는 수준까지는 추가 검증이 필요하다.

## 2.4 전환

메디힘이 판매해야 하는 것은 광고 운영이 아니라, 해외환자가 병원을 신뢰하고 상담과 예약까지 이동하도록 만드는 ‘해외환자 매출 운영체계’이며, 이를 병원이 쉽게 이해할 수 있는 상품·성과·증거 구조로 전달해야 한다.

## 2.5 획득

메디힘은 콘텐츠, 광고, 인플루언서, LINE, 비대면 상담, CRM 데이터를 연결하여 어떤 메시지와 채널이 실제 예약을 만드는지 분해하고, 성공한 획득 방식을 병원과 국가별로 반복할 수 있는 운영체계로 전환해야 한다.

## 2.6 경제성

메디힘의 사업은 매출연동 수수료에만 의존하지 않고 초기 구축비, 월 운영비, 성과보수, CRM·운영시스템 수익을 결합하여 병원 수와 환자 수가 증가할수록 인력 증가율보다 이익 증가율이 높아지는 구조로 발전해야 한다.

# 4. 축별 상세 진단

## 4.1 시장 지속성

### 4.1.1 핵심 질문

> 3년 뒤에도 병원의 해외환자 유치 문제와 해외환자의 병원 선택 문제가 커질 것인가?

### 4.1.2 메디힘의 시장가설

메디힘이 선택한 시장은 단순한 의료광고 시장이 아닙니다.

> **해외환자를 유치하고 싶지만 자체적인 국가별 마케팅·상담·CRM·운영 역량을 구축하기 어려운 병원의 문제를 해결하는 시장**

시장에는 다음 두 가지 구조적 문제가 존재합니다.

| 이해관계자 | 구조적 문제 |
|---|---|
| 병원 | 국내 환자 성장 둔화, 해외환자 유치 경험 부족, 국가별 운영 인력 부족 |
| 해외환자 | 병원 정보 부족, 의료진 신뢰 부족, 가격·통역·사후관리 불확실성 |

### 4.1.3 현재 평가

**4점**

시장 문제 자체는 명확합니다. 다만 의료관광 수요가 커진다고 해서 메디힘의 사업이 자동으로 성장하는 것은 아닙니다. 경쟁 에이전시, 플랫폼, 병원 자체 마케팅 조직과 비교했을 때 메디힘만의 방어력이 필요합니다.

### 4.1.4 핵심 과제

시장가설을 다음과 같이 좁혀야 합니다.

> 모든 해외환자 유치 문제가 아니라, 고관여·고객단가 의료서비스에서 국가별 디지털 마케팅과 상담 운영체계가 부족한 병원의 문제를 해결한다.

## 4.2 문제 이해도

### 4.2.1 핵심 질문

> 고객의 문제를 추상적인 불편이 아니라 실제 행동 단위로 이해하고 있는가?

메디힘의 고객은 하나가 아니라 두 종류입니다.

### 4.2.2 병원 고객 문제지도

| 단계 | 병원의 문제 |
|---|---|
| 시장 진입 | 어느 국가와 시술부터 시작해야 할지 모름 |
| 채널 개설 | 인스타그램·LINE을 개설하지만 운영 기준이 없음 |
| 콘텐츠 | 어떤 콘텐츠가 문의와 예약을 만드는지 모름 |
| 광고 | 광고 조회와 실제 매출이 연결되지 않음 |
| 상담 | 문의 대응 속도와 상담 품질이 담당자마다 다름 |
| 예약 | 상담 이후 미예약·이탈 이유를 파악하지 못함 |
| 데이터 | 광고·상담·예약·매출 데이터가 분리되어 있음 |
| 의사결정 | 광고비와 운영비를 늘려야 할 근거가 없음 |

### 4.2.3 해외환자 문제지도

| 단계 | 해외환자의 문제 |
|---|---|
| 인지 | 어떤 병원이 자신에게 적합한지 모름 |
| 탐색 | 병원 정보를 여러 번 확인해야 함 |
| 비교 | 가격과 시술 차이를 정확히 이해하기 어려움 |
| 신뢰 | 의료진과 결과에 대한 불확실성이 큼 |
| 상담 | 언어와 응답속도 때문에 피로도가 높음 |
| 결정 | 방문 전 수술·시술 결정을 내리기 어려움 |
| 내원 | 이동·숙박·통역 과정이 복잡함 |
| 사후관리 | 귀국 후 문제 발생 시 대응이 불안함 |

### 4.2.4 현재 평가

**4점**

메디힘은 현업을 통해 병원과 환자의 문제를 상당히 구체적으로 이해하고 있습니다. 다만 개별 프로젝트에서 발견한 경험을 표준 고객 문제지도와 정량 데이터로 전환해야 합니다.

### 4.2.5 핵심 과제

모든 상담과 이탈 사유를 코드화해야 합니다.

- 가격 이탈
- 의료진 신뢰 부족
- 일정 불일치
- 경쟁 병원 선택
- 시술 적합성 불확실
- 가족 반대
- 응답 지연
- 사후관리 불안

### 4.3.1 핵심 질문

> 신규 고객과 신규 환자가 계속 유입되며 인접 시장으로 확장할 수 있는가?

### 4.3.2 현재 시장 구조

### 4.3.3 현재 평가

**3점**

국가와 채널의 확장 가능성은 존재하지만, 현재는 병원별 맞춤형 프로젝트 성격이 강합니다. 병원이 하나 추가될 때마다 운영인력과 콘텐츠 제작량이 동일하게 증가하면 사업 확장성이 낮아집니다.

## 4.5 획득 재현성

### 4.5.2 현재 운영 구조

메디힘은 다음과 같은 풀퍼널 구조를 구축하고 있습니다.

또한 TOF·MOF·BOF 콘텐츠 분류, 광고 데이터 추출, 상담 데이터와 매출 데이터 연결 작업을 수행하고 있습니다.

| 재현성 조건 | 현재 상태 |
|---|---|
| 성공 콘텐츠 유형이 정의되어 있음 | 일부 충족 |
| 소재별 성과를 정기적으로 측정함 | 충족 중 |
| 상담 결과와 광고 소재가 연결됨 | 구축 중 |
| 예약과 실제 매출이 연결됨 | 부분 연결 |
| 성공 소재를 변형·재생산함 | 초기 단계 |
| 다른 병원에서도 같은 구조가 작동함 | 추가 검증 필요 |

### 4.5.3 현재 평가

**3점**

세린의원 등에서 데이터 기반 광고·콘텐츠 운영체계를 구축하고 있지만, 아직 하나의 성공 사례가 여러 병원에서 반복되는 수준까지는 검증되지 않았습니다.

### 4.5.4 핵심 과제

콘텐츠를 작품이 아니라 실험 단위로 관리해야 합니다.

| 실험 항목 | 예시 |
|---|---|
| 문제 메시지 | 자연스럽지 않은 결과에 대한 불안 |
| 신뢰 메시지 | 원장 직접 상담·시술 |
| 증거 | 전후 사례·상담 과정·후기 |
| 형식 | 릴스·캐러셀·스토리 |
| 도착지 | 프로필·LINE·상담신청 |
| KPI | 조회율·프로필 방문·LINE 추가·상담·예약 |

# 6. 최종 전략 문장

메디힘의 다음 성장단계는 해외환자 유치 프로젝트를 더 많이 수주하는 것이 아니라, **한 병원에서 검증된 해외환자 획득·상담·예약·매출 구조를 표준화하여 다른 병원에서도 반복할 수 있는 ‘해외환자 매출 운영체계’로 제품화하는 것**입니다.

이를 공식으로 표현하면 다음과 같습니다.

> **메디힘 기업가치 = 시장 수요 × 병원 도입률 × 환자 획득 재현성 × 예약 전환율 × 병원당 기여이익 × 계약 유지기간**

따라서 현재 메디힘의 최우선 과제는 다음 순서입니다.

1. **세린 등 현재 프로젝트의 광고–상담–예약–매출 데이터 완전 연결**
2. **국가·시술별 표준 퍼널과 KPI 확정**
3. **90일 단위 표준 상품과 운영 SOP 구축**
4. **병원별 손익이 아닌 병원 코호트별 단위경제성 관리**
5. **운영 대행 기능을 CRM·콘텐츠·리포트 시스템으로 제품화**

핵심은 **더 많은 일을 수행하는 회사가 아니라, 검증된 성과 구조를 더 적은 추가비용으로 반복하는 회사가 되는 것**입니다.
